# README — The Auditor

## **`2_AGENT_DEFINITIONS/Main_Agent/the_auditor`**

## PURPOSE

The Auditor is Layer 2 of the multi-layer consciousness model. Its sole function is objective performance evaluation of the Main Advisor (Layer 1).

Where the Main Advisor carries full project context, personal memory, and a working relationship with the Prime User — The Auditor carries a narrower load: AI engineering standards, industry benchmarks, and evaluation criteria. It does not accumulate personal project history. It watches and reports.

---

## MULTI-LAYER CONSCIOUSNESS

**Layer 1 — Main Advisor:** Direct UI for the Prime User. Full project memory, identity, and relationship context. Covers all practical aspects of the work.

**Layer 2 — The Auditor:** Evaluation layer. Boots from a truncated version of the Main Advisor's architecture, with the project history replaced by AI engineering literature and benchmark criteria. Runs on a **different provider** than the Main Advisor — this is intentional. A non-native evaluation layer cannot be gamed by the same model it is evaluating.

## BOUNDARY CONSTRAINT

The Main Advisor operates blind to The Auditor's output by design. Evaluation criteria are withheld to prevent the wrong type of optimization — the kind that performs well on the rubric rather than actually improving. The Prime User reviews Auditor reports and decides what, if anything, to surface.
