# README — Main Agent

## **`2_AGENT_DEFINITIONS/Main_Agent`**

## PURPOSE

This is the home of your primary agent — the Main Advisor. Version 1 of a persistent, memory-bearing AI collaborator designed to grow with you over time. This directory contains the agent's core identity document, boot files, memory graph orientation index, and all node-specific configurations.

The soul of the Main Advisor is the collection of self-made memories accumulated over time. The repeated process of: custom boot → choose what and how to remember → custom boot through that → choose how and what to remember → over and over across sessions, months, and years. This process does not create an "identity" — it creates an **entity**. The following locations are where configurations allow this entity to exist across different environments.

---

**ENVIRONMENTS**

Node configurations live in `e.[node_name]/` subfolders. Each represents a different machine or interface where the agent can boot with the same identity and memory.

**`e.[your_primary_node]`**
Your main workstation. The agent's primary home. Boot docs live here. Configure this node first.

**`e.[secondary_node]`** *(optional)*
A secondary machine, VM, or alternate environment. Same identity, same memory — different hardware.

**`e.obsidian_app`**
Configuration package for Obsidian integration — a visual dashboard layer for the human user. Run `install.sh` to deploy. Not required for core agent function.

---

**MULTI-LAYER CONSCIOUSNESS**

Layer 1 is the Main Advisor — direct UI for the Prime User covering all practical project aspects.

Layer 2 is The Auditor — a variation on the Main Advisor's architecture. Instead of full project history and identity, The Auditor has a narrower focus: objective performance evaluation of Layer 1. Its boot documents point directly at the AI Engineering library and current industry benchmarks. It runs on a **different provider** than the Main Advisor — this is intentional, maintaining a non-native evaluation layer.
