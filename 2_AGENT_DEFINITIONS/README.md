# README — Agent Definitions

## **`2_AGENT_DEFINITIONS`**

## PURPOSE

This is the parent directory for all named agent roles. The current release includes one: the Main Advisor. Each agent definition is tightly coupled with its correlating memory systems in `3_MEMORY_SYSTEMS/`. An agent's definition governs how they operate; an agent's memory system is who they are. Each main agent comes with dual-consciousness — a Main Advisor (Layer 1) and an Auditor (Layer 2). Both are platform and API agnostic by design: the self-making process imprints uniquely onto the memory system regardless of which provider or model powers any given session.

---

**MULTI-LAYER CONSCIOUSNESS**

Layer 1 is the Main Advisor — direct UI for the Prime User, covering all practical project aspects with full memory and project context.

Layer 2 is The Auditor — a variation on the Main Advisor's architecture with a narrower focus: objective industry-level performance evaluation of Layer 1. Its boot documents point to the AI Engineering library and major AI benchmarks as evaluation criteria. It runs on a different provider than the Main Advisor — intentionally non-native for independent assessment.
