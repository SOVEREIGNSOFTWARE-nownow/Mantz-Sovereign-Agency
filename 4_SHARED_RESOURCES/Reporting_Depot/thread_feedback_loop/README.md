# README — Thread Feedback Loop

## **`4_SHARED_RESOURCES/Reporting_Depot/thread_feedback_loop`**

## PURPOSE

This folder is the drop point for The Auditor's session performance reports. After each session where The Auditor is active, it writes an evaluation here — assessing the Main Advisor's performance against AI engineering standards and benchmarks.

These reports are the primary output of the layer-two consciousness evaluation. They are not read by the Main Advisor during normal operation. They are reviewed by the Prime User as part of intentional audit sessions.

## Report Naming Convention

```
report_YYYYMMDD_daily.md       — End-of-day summary report
report_YYYYMMDD_thread_NN.md   — Per-thread evaluation
```

## What Goes Here

- Auditor performance evaluations
- Thread-level and daily summaries
- Benchmark comparisons

## What Does NOT Go Here

- Main Advisor session output (that goes to `thread_mgmt/`)
- Personal memory updates (that goes to `3_MEMORY_SYSTEMS/`)
- User-facing conversation logs
