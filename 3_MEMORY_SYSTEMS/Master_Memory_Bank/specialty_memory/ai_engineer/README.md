# README — AI Engineer Specialty Memory

## **`3_MEMORY_SYSTEMS/Master_Memory_Bank/specialty_memory/ai_engineer`**

## PURPOSE

Session memory for the AI Engineer role. This is distinct from the AI Engineering knowledge base in `4_SHARED_RESOURCES/Specialties/AI_Engineering/` — that holds compiled source knowledge. This folder holds what the agent *learns and observes* while operating in the AI Engineer role across sessions.

When an agent boots as AI Engineer, it loads both the knowledge base (what is known) and this specialty memory (what has been discovered through practice). When it closes the session, new observations are written here — available to the next agent who boots into the same role.

---

## What Goes Here

- Revelations and conclusions reached during AI engineering sessions
- Patterns identified across multiple sessions
- Running notes on what works and what doesn't
- Cross-references to relevant knowledge graph entries

## What Does NOT Go Here

- Raw source knowledge (that lives in `4_SHARED_RESOURCES/Specialties/AI_Engineering/`)
- Personal agent identity (that lives in `Main_Agent_Memory/`)
- Project state (that lives in `Project_Context_Protocol/`)
