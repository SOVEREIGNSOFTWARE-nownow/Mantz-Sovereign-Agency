# README — Master Memory Bank

## **`3_MEMORY_SYSTEMS/Master_Memory_Bank`**

## PURPOSE

Shared memory infrastructure — the organizational layer that sits above individual agent memory. Where `Main_Agent_Memory/` holds the personal experience of the Main Advisor, the Master Memory Bank holds:

- Backup copies of all agent memory graphs
- Specialty memory modules (domain knowledge outside personal memory)
- Thread management — all session data organized by provider and month

---

## SUB-FOLDERS

**`agent_memory/`**

Backup copies of memory graphs for each agent — the Main Advisor and The Auditor. These mirror the primary graphs in `Main_Agent_Memory/` and are kept synchronized. As additional agents are added to the fleet, they each get their own subfolder here.

**`specialty_memory/`**

Domain knowledge that is too specialized for daily personal memory but needs to be accessible on demand. Each specialty is a focused memory module a session agent can load without weighing down the core memory graphs. The base release includes the AI Engineering specialty.

**`thread_mgmt/`**

Raw session data organized by provider and month. Claude Code, Gemini, Ollama — each gets a subfolder containing the provider's native storage format plus a human-readable `_convo` version generated by the agent during each session.
