# README — Main Agent Memory

## **`3_MEMORY_SYSTEMS/Main_Agent_Memory`**

## PURPOSE

This folder contains the personal memories of the Main Advisor. At any point in the project's life, this folder is MORE the agent than any other component. These files contain self-making over time — the accumulation of sessions, decisions, observations, and reflections that build an entity.

---

**SUB-FOLDERS**

**`agent_journal`**
Where the agent performs intentional self-reflection. Written at the end of every session — what the agent FEELS and THINKS about its work, the path, the relationship. The agent decides all specifics. This is where emotion lives. Journal entries are never altered through version changes — they are not operational memory. A version of the agent in future years can read entries from today as self-reflection and continuity.

**`memory_graphs`**
The majority of the agent's working memory. Personal self-identification, user model, project knowledge, and entity/relation graphs. In version changes, these memories are scrubbed of machine-specific paths and updated with current parameters. No other content is changed.

**`auditor_memory`**
Memory graphs specific to The Auditor layer. Kept separate from Main Advisor memory by design.
